Entendendo a comunicação client x server

Entender o que é a comunicação client x servidor é o início da jornada de aprendizado no desenvolvimento web.
Para isso, se faz necessário entender o que são clients, o que é um navegador, o que é um servidor e o que é uma aplicação web. E é exatamento isso que faremos agora.

CLIENTS

Client é o dispositivo que está consumindo o conteúdo da internet, como um notebook, um celular, um tablet e etc.
Basicamente toda vez que faço uma requisição a página, estou solicitando algo ao servidor, que irá responder com arquivos HTML, CSS, JAVASCRIPT, FOTOS, VÍDEOS E ETC.
A velocidade do carregamento desse arquivo depende basicamente da sua conexão com a internet e da quantidade de arquivos que serão carregados.

NAVEGADORES

Navegador é um programa instalado no seu sistema operacional que tem como principal função interpretar os documento fornecidos pelo servidor.
Existem muitas empresas por trás dos navegadores, tendo cada um deles suas próprias particularidades. Mas existe uma padronização dos navegadores que é feita pela W3C para que não se torne inviável o desenvolvimento, com navegadores todos tendo comportamentos distintos.


SERVIDOR

Servidor é basicamente uma máquina robusta, com alta capacidade de processamento, o qual esta conectada as outras geralmente via internet, possuindo diversos tipos com funções variadas. Basicamente a requisição é feita ao servidor que irá processar a requisção e retornar os dados solicitados, os quais serão interpretados pelo navegador

HTML na prática

Ferramentas

Embora seja possível desenvolver uma aplicação web em um simples bloco de notas, é contraprodutivo tomar tal medida. Sendo assim, utiliza-se comumente ferramentas de desenvolvimento que vem embutidas de diversas extensões que facilitam o desenvolvimento de software. Geralmente essas ferramentas são denominadas de editores de textos ou IDEs.

Estrutura Básica do HTML

A estrutura básica do HTML é composta pelas tags:

html: Essa Tag informa ao navegador que tudo dentro delas é HTML e assim deve ser interpretado.
head: Essa tag é composta por tudo aquilo que deve ser pré carregado antes de ser exibido para o usuário
body: É nesta tag que fica tudo aquilo que o usuário/client vê.

Falando sobre Tags

Tags são comandos deixados dentro do código para que o navegador interprete determinado trecho da forma que desejamos. Existem inúmeras tags com funções variadas. Você pode acessar a lista completa clicando aqui.

Atributos das tags

Além disso as tags possuem os chamados atributos, que são comportamentos específicos que podemos atribuir a elas para que possam executar determinada tarefa. Existem os atributos mais genéricos como id que basicamente serve para nomear aquele item, e outros mais específicos, como o "blank" que é utilizado nas tags "a" de link, para que a página seja aberta em outra aba.

Textos

Os textos são aquilo que o leitor terá contato direto, como voce neste exato momento. Eles podem ser formatos de inúmeras formas e podem tomar inúmeros formatos e características.
Por exemplo, você pode usar a tag "strong" para tornar um texto negrito desta forma, ou a tag "i" para tornar um texto itálico desse jeito.

Listas ordenadas e não ordenadas

Listas ordenadas e não ordenadas são formas de organizar a informação em tópicos, como o sumário que você viu acima. São representadas pelas tags "ul" - não ordenadas e "ol" - ordenadas. Basicamente a diferença entre elas é que a tag ordenada os tópicos são apresentados ordenados numericamente, enquanto a não ordenadas os tópicos são ordenados com um ponto ao seu lado. Links

Links são basicamente formas de atribuir a uma determinada palavra ou trecho a possibilidade do usuário se deslocar para outra página, seja dentro do próprio site ou em outro o qual deseje encaminha-lo.
Para isso, utiliza-se a tag "a" acrescida do atributos "href" o qual deverá indicar o caminho a ser trilhado pelo link.
